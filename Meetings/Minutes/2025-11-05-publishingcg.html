Meeting minutes

Breno: Thanks for the invitation, I have a presnetation to share.
… NEES is a research group located in Maceios, Brazil.
… we work a lot in education equity and digital transformation, often with administration and in link with public policies.
… Brazil's National Textbook Program (PNLD) is about 180k schools for which more than 10 publishers provide textbooks.
… The programme is managed by Ministry of education and FNDE (national fund for the developement of education. It is a very large program
… it provides free high quality education material.
… covers all stages of education
… recently it has been expanded to digital formats aligning with national policies on inclusive education.
… digital books is a part, but there is also a need to deploy, meaning we need a reading system. That's the task of my group.
… We develop the multiplateform reader and define format to be produced.
… we have publishers, pedagogical evaluators, technical anlysts, accessibility analysts
… and ressource analysts all of that to approve ressources and make sure they are high quality and accessible.
… that means both pedagogical and technical analysis and quality assurance.
… ebooks include multimedia.
… it's a complex sequential process to make a textbook first, usually produced as PDF, summmited for pedagogical aspects, then if accepted, it will be converted as digital and enhanced with multimedia. At the end, the digital version is completely different, we have to re assess the pedagogical aspects.

Breno: the digital version is the one that comes with full accessibility requirement, it has to be very well structured, standard compliant but also include accessibility for multimedia contents (video, audio, forms).
… given the complexity of the publications, this phase is crucial. The price negociation phase comes after.
… Then if an agrement is found, comes production and distribution.
… Analysts already use the reading system, I mean the same as the one, that is used by students.
… So our reading system is both used to study and to assess quality of the ebook.
… This production process is long, we know it could be made simpler if a digital versin was developped previously to the printed version.
… Once ebooks produced, then we need to deliver it to end users. We have connectivity issues in some parts of Brasil, including instable and bad connectivity. We have to deal with that scenario. Remeber that the ebooks include audio and video.
… we also have the protection concerns and need to use DRM.
… We want to have more interactivity like being able to perform exercices in the textbook.
… we built a e-textbooks taxonomy, published by edulearn in 2025. From digitized to intelligent passing by multimedia, interactive and immersive.
… That leads us to being interested in web publications
… the standard is not much updated, but we met EDRLab and understood how to use this format following up readium implementations.
… As the tools are open source we've been able to create a viable implementation
… with the help of the readium developers.
… we are working on tooll to ease publishers transition to this format to be able to fully use it along the described chain, from publishing to reading.
… so HTML5 is the creation format, we convert it today as EPUB but face restrictions on javascript and CSS3 which is not a burden anymore if we use Web Publication.
… also to mention, before we were using our own format, we have moved to standards and wish to follow up).
… The main move for us is to shift from OPF to JSON, then HTML files with no restrictions, that's great.
… Web publication also ease the partial download which is important in our context with connectivity issues.
… It brings the best of Web and EPUB together in our view, and Readium implementation eases the onboarding for our dev teams, and for publishers.

… as challenges, we see a difficulty related to consistent rendering bvbetween plateform, but it's much better as web publication lower the need of code injection on the reading system side.
… connectivity and distribution is partially solved, but we also think about P2P content centric delivery (ipfs.tech)
… most of all, the move to web publication allows us to ease the creation of accesisble and interactive ebooks by publishers.
… are there questions?

wolfgang: with interactivity, digital publishing is going forward, it means escaping from the paper reproduction. It allows for immersion, fun and participation. I think it is made for not all ebooks, not all publishers. It is good to see ebooks toughts as digital objects.
